# 任务协议（TASK PROTOCOL）
> 通用任务拆解与执行协议。
> 与项目无关，适用于任何收到的任务。

---

## 动手之前

1. 读 `HARNESS.md` — 确认你已知晓不变量和子系统地图。
2. 读 `BOOTSTRAP.md` — 确认你已知晓入口文件和开发命令。
3. 判断任务涉及哪个子系统。
4. 只读负责该子系统的 2–3 个文件。不要通读整个代码库。

---

## 四个阶段（必须按此顺序执行）

### 阶段 A — 理解

目标：在改任何东西之前，彻底搞清楚代码当前的行为。

- [ ] 找到相关的类型 / 接口定义
- [ ] 从入口点追踪一条真实的执行路径，直到要修改的代码
- [ ] 确认你将要改动的函数 / 模块的所有调用方和被调用方
- [ ] 确认当前行为（读代码，不要猜测）

**如果任务描述与代码实际行为矛盾，在这里停下**，寻求澄清，不要靠猜继续。

### 阶段 B — 计划

目标：动文件之前，先把要做的事情写下来。

- [ ] 列出所有需要修改的文件
- [ ] 列出所有需要新增的类型 / 接口
- [ ] 识别必须保持不变的不变量（见 `HARNESS.md §「不变量」`）
- [ ] 判断影响范围：如果做错了，会有什么后果？
- [ ] 确定实现顺序（见下方「执行顺序规则」）

**把计划写成简短的要点列表。** 如果超过 10 条，说明任务太大——拆分它。

### 阶段 C — 实现

目标：满足任务要求的最小改动量。

按以下顺序执行：

1. **先改类型 / 接口** — 在改实现之前，先改类型定义。
   调用方会在编译时报错，让你立刻看清全部影响范围。
2. **再改核心逻辑** — 实现函数、类或模块。
3. **最后做注册 / 接线** — 往注册表、索引文件或主配置里添加条目。
   这一步永远是最后一步。

不要添加任务以外的功能。
不要重构你本不需要改动的代码。
不要为不可能发生的情况添加错误处理。

### 阶段 D — 验证

目标：在宣布完成之前，确认改动有效。

- [ ] 运行冒烟测试命令（来自 `BOOTSTRAP.md`）
- [ ] 手动执行受影响的路径至少一次
- [ ] 过一遍 `REVIEW.md` 检查清单

---

## 执行顺序规则

这些规则在阶段 C 内部适用，优先于直觉判断：

**1. 类型先于实现。**
如果你在引入一个新概念，先定义它的类型。编译器会成为你的依赖关系图——它会告诉你所有需要更新的地方。

**2. 核心逻辑先于接线。**
在注册之前，先在独立环境中实现逻辑。一个还没注册的工具仍然可以测试；一个已注册但有 Bug 的工具会让整个系统崩溃。

**3. 单文件改动先于跨文件改动。**
尽量把改动限定在一个文件中。只有在编译器或测试强制要求时，才扩大范围。

**4. 永远不要破坏公共契约。**
如果其他代码依赖某个函数签名或返回类型，除非任务明确要求，否则不要修改它。改用新增函数的方式。

**5. 垫片 / 兼容层文件只读。**
如果项目有垫片或兼容层，除非任务明确针对这些文件，否则不要编辑它们。如有需要，绕过它们。

## 特定任务类型的执行模式

> 【本节与项目相关，安装 Harness 时填写。
>  下面的示例仅作说明用途，请替换成你的项目的实际模式。】

### 模式：新增一个能力单元
> 【例：工具、服务、处理器、控制器、插件】

```
1. 定义接口 / 类型（src/types/ 或对应目录）
2. 创建实现文件（src/<模块>/<名称>.ts）
3. 实现必须有的方法：【列出方法名】
4. 注册到中央注册表：【注册文件路径 + 怎么添加】
5. 验证：【如何测试这种类型】
```

### 模式：新增一个用户可见的命令
> 【例：CLI 命令、API 端点、斜杠命令】

```
1. 定义输入 / 输出类型
2. 实现处理逻辑（保持处理器薄——将重逻辑下沉到服务层）
3. 在命令路由器 / 路由文件中注册：【路径】
4. 验证：【如何调用和测试】
```

### 模式：修改核心执行流程
> 【例：查询循环、请求管道、事件总线】

```
1. 先从头到尾读完流程文件——异步生成器 / 流式处理很微妙
2. 判断你要改的是哪个阶段（初始化 / 执行 / 收尾）
3. 不要破坏 yield / 回调契约——调用方依赖事件的顺序
4. 验证：跑完整的端到端流程，不要只跑单元测试
```

## 完成标准

满足以下全部条件，任务才算完成：

- [ ] 冒烟测试 exit 0
- [ ] 手动执行受影响的路径，结果正确
- [ ] 没有新增编译器 / 类型错误
- [ ] `REVIEW.md` 检查清单全部通过
- [ ] 输出写到了 `IO_MAP.md` 规定的正确位置
- [ ] 阶段 B 制定的计划已全部执行（没有跳过任何步骤）
