# 智能体工作台（HARNESS）
> 通用 Harness Engineering 模板。
> 将此文件夹复制到任何项目中，填写括号内的内容。
> 没有任何对话上下文的全新智能体，按顺序读完这几个文件后即可立刻开始工作。

---

## 阅读顺序

```
1. HARNESS.md          ← 你在这里。决策树 + 关键类型速查。
2. BOOTSTRAP.md        ← 这是什么项目，代码在哪里
3. TASK_PROTOCOL.md    ← 如何拆解任务、按什么顺序推进
4. IO_MAP.md           ← 从哪里读输入，往哪里写输出
5. （执行工作）
6. REVIEW.md           ← 标记完成前的检查清单
```

---

## 决策树

收到任务后，先用这棵树定位，再去读 BOOTSTRAP.md 里对应的子系统。

```
收到任务 →

  这是 Bug 修复？
    → 找到出问题的执行路径。读 IO_MAP.md §「从哪里读」。
      然后读 TASK_PROTOCOL.md §「阶段 A：理解」。

  这是新功能？
    → 完整执行 TASK_PROTOCOL.md（四个阶段全走）。

  这是重构 / 清理？
    → 先读涉及的文件，再套用 TASK_PROTOCOL.md §「执行顺序规则」。

  这是代码审查 / 审计？
    → 直接跳到 REVIEW.md。

  不确定？
    → 读 BOOTSTRAP.md §「子系统地图」，确认任务归属，再重新从这里开始。
```

## 不变量 — 本项目永远成立的规则

> 【按项目填写。这些是全新智能体必须在开始工作前就知道的硬约束。】

```
不可变性：    【例：状态对象永远不可原地修改，必须使用 setX(prev => ...) 模式】
并发性：      【例：后台任务必须幂等，禁止访问共享可变全局变量】
权限控制：    【例：所有用户输入在进入数据库或文件系统前必须经过校验】
输出约定：    【例：任务输出只能追加写入磁盘文件，不允许持久保存在内存中】
代码风格：    【例：TypeScript strict 模式，禁止 any，仅使用 ESM 导入】
```
