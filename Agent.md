**多 Agent 协作 + 自动闭环开发流程**

------

# 一、核心机制：多 Agent 分工协作

至少有 3 个 Agent：

### 1️⃣ 系统设计 Agent

文件：`系统设计.md`
职责：

- 定义架构
- 定义模块边界
- 定义接口规范（API / 数据结构）
- 输出开发约束

👉 本质：**架构师**

------

### 2️⃣ 开发 Agent

文件：`开发.md`
职责：

- 根据系统设计实现代码
- 完成功能开发
- 修复缺陷

👉 本质：**工程师（可自迭代）**

------

### 3️⃣ 测试 Agent

文件：`测试.md`
职责：

- 执行测试（可能是自动测试 / 用例驱动）
- 发现问题
- 反馈给开发 Agent

👉 本质：**QA + 自动验证器**

------

# 二、自动化流程（关键！）

整个流程是一个**闭环循环系统**：

```
需求.md
   ↓
系统设计 Agent
   ↓
开发 Agent
   ↓
测试 Agent
   ↓
（如果失败）→ 回到开发 Agent 修复
   ↓
（循环）
   ↓
最终通过
```

👉 这是一个典型的：

> 🔁 **Self-Refining Agent Loop（自我修复循环）**

------

# 三、关键技术实现方式

这种系统通常是这样实现的：

------

## 1️⃣ Prompt 编排（最核心）

每个 `xxx.md` 实际上是：

👉 一个**超强 Prompt 模板**

例如：

### 系统设计.md（伪代码）

```
你是系统架构师

输入：
- 需求文档

输出：
- 模块设计
- API定义
- 数据结构
- 技术选型
```

------

### 开发.md

```
你是工程师

输入：
- 系统设计
- 当前代码
- 缺陷列表

输出：
- 修改后的代码
```

------

### 测试.md

```
你是测试工程师

输入：
- 当前代码
- 需求

输出：
- 是否通过
- 错误列表
```

------

👉 本质：
**用 Prompt 约束 LLM 扮演不同角色**

------

## 2️⃣ Agent 调度器（Orchestrator）

需要一个“调度程序”来控制流程：

### 伪代码

```python
while True:
    design = run_agent("系统设计", requirement)

    code = run_agent("开发", design)

    result = run_agent("测试", code)

    if result.pass:
        break
    else:
        code = run_agent("开发", result.errors)
```

👉 这个就是：

> 🧠 **Agent Loop Engine**

常见实现方式：

- Python + LangChain / LangGraph
- 自己写状态机
- 或用 AutoGen / CrewAI

------

## 3️⃣ 状态存储（关键点）

系统需要存这些内容：

- 当前代码
- 设计文档
- 测试结果
- 错误日志

👉 通常用：

- 本地文件（最简单）
- 或向量数据库（做记忆）
- 或 Git（最强方案）

------

## 4️⃣ 自动反馈机制（闭环关键）

测试 Agent 输出结构类似：

```json
{
  "pass": false,
  "errors": [
    "接口返回字段不符合规范",
    "边界条件未处理"
  ]
}
```

然后喂给开发 Agent：

👉 实现：

> 🔁 **自动 Debug Loop**

# 五、它的“高级点”在哪里？

这段设计其实有几个很聪明的点👇

------

## ⭐ 1. 用文件分角色（很工程化）

不是写死在代码里，而是：

```
agents/
 ├── 系统设计.md
 ├── 开发.md
 ├── 测试.md
```

👉 好处：

- 可扩展（加 Agent 很容易）
- 可调 Prompt
- 可复用

------

## ⭐ 2. 自动触发测试

```
开发完成 → 通知测试
```

👉 这其实是：

> 📦 **事件驱动（Event-driven）**

------

## ⭐ 3. 自动修复循环

```
测试失败 → 开发修复 → 再测
```

👉 核心价值：

> 🚀 **无人值守开发（Autonomous Dev Loop）**

------

## ⭐ 4. 目标驱动（Goal-driven）

最终目标：

```
符合 《需求.md》
```

👉 这就是：

> 🎯 Goal-based Agent
